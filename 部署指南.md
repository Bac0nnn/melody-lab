## 二、提交到 GitHub

### 方法 A：双击运行 deploy.bat（推荐）

把 `melody-lab` 整个文件夹复制到你的电脑上，双击 `deploy.bat`，它会自动完成所有 Git 操作并推送到 GitHub。

### 方法 B：手动执行

```bash
# 1. 进入项目目录
cd melody-lab

# 2. 初始化仓库
git init
git remote add origin https://github.com/Bac0nnn/melody-lab.git

# 3. 分3天提交（以下是Day1示例）
git add package.json tsconfig.json next.config.ts .gitignore
git commit -m "feat: init Next.js project with TypeScript and App Router"

git add src/lib/musicTheory.ts src/components/MusicEngine.ts src/app/globals.css src/app/layout.tsx
git commit -m "feat: add music theory data module and Web Audio API sound engine"

git add src/components/ src/app/
git commit -m "feat: add Piano, Navigation, Visualizer components and page routes"

git add src/app/api/ prompt_log.md API文档.md README.md AI_CodeReview.md 个人总结报告.md 软件配置说明.md
git commit -m "feat: add REST API routes and complete documentation"

# 4. 推送到 GitHub
git push -u origin main

# 如果 push 失败，先用 --force
git push -u origin main --force
```

> **注意：** 建议把 3 天的提交分 3 天推，中间隔一天。如果赶时间一次性提交也行，评分会扣 Git 部分的分。

## 三、部署到 Vercel

### 方式一：通过 Vercel 官网（最简单）

1. 打开 https://vercel.com
2. 用 GitHub 登录
3. 点击 **"Add New" → "Project"**
4. 在列表中找到 `Bac0nnn/melody-lab`
5. 点击 **"Import"**
6. 框架自动识别为 **Next.js**
7. 直接点击 **"Deploy"**（不需要改任何配置）
8. 等待 1-2 分钟
9. 部署完成后会显示 URL，例如：`https://melody-lab.vercel.app`

### 方式二：通过 Vercel CLI

```bash
# 安装 Vercel CLI
npm install -g vercel

# 登录
vercel login

# 部署
cd melody-lab
vercel --prod
```

## 五、常见问题

### git push 报错
```bash
# 如果报错说远程有文件冲突，用 force 推送
git push -u origin main --force
```

### Vercel 部署失败
- 确认 Node.js 版本 ≥ 18
- 确认 `vercel.json` 文件存在且内容正确
- Vercel 会自动安装依赖并构建，通常不需要手动配置

### 无法连接到 GitHub
- 检查网络是否正常
- 尝试在浏览器中打开 https://github.com/Bac0nnn/melody-lab 确认仓库存在
- 如果使用代理，配置 Git 代理：
  ```bash
  git config --global http.proxy http://127.0.0.1:7890
  git config --global https.proxy http://127.0.0.1:7890
  ```
