# 软件配置说明 — Melody Lab

## 一、所需软件清单

| 软件 | 版本要求 | 用途 | 下载地址 |
|------|---------|------|---------|
| Node.js | v18+ | JavaScript 运行时 | https://nodejs.org/ |
| VS Code | 最新版 | 代码编辑器 | https://code.visualstudio.com/ |
| Git | v2+ | 版本控制 | https://git-scm.com/ |
| Chrome/Edge | 最新版 | 浏览器调试 | 系统自带 |
| Vercel CLI | 最新版 | 部署上线 | `npm i -g vercel` |

## 二、安装与配置

### 1. 安装 Node.js
1. 访问 https://nodejs.org/ 下载 LTS 版本（v18+）
2. 运行安装程序，一路 Next
3. 验证安装：
```bash
node --version   # 应显示 v18.x.x 或更高
npm --version    # 应显示 9.x.x 或更高
```

### 2. 安装 VS Code
1. 访问 https://code.visualstudio.com/ 下载安装
2. 推荐安装插件：
   - **ESLint** — 代码规范检查
   - **Prettier** — 代码格式化
   - **Next.js snippets** — Next.js 代码片段
   - **GitLens** — Git 历史可视化

### 3. 安装并配置 Git
1. 访问 https://git-scm.com/ 下载安装
2. 配置用户信息：
```bash
git config --global user.name "你的名字"
git config --global user.email "你的邮箱@example.com"
```

### 4. 配置 GitHub 仓库
```bash
# 在 GitHub 新建仓库后执行
git init
git add .
git commit -m "feat: initial commit - Melody Lab music theory tool"
git branch -M main
git remote add origin https://github.com/你的用户名/melody-lab.git
git push -u origin main
```

### 5. 每天提交示例（满足考核 3 天要求）
```bash
# Day 1 - 项目初始化
git commit -m "feat: scaffold Next.js project with TypeScript"
git commit -m "feat: add music theory data module (chords, scales, quiz)"

# Day 2 - 核心功能
git commit -m "feat: implement Web Audio API MusicEngine"
git commit -m "feat: add interactive Piano component with highlight support"
git commit -m "feat: create Explore page for chord/scale discovery"

# Day 3 - 完善与部署
git commit -m "feat: add Canvas visualizer with 3 visualization modes"
git commit -m "docs: add README, API docs, prompt log, and summary report"
git commit -m "fix: improve piano black key positioning and responsive layout"
```

### 6. 配置 Vercel 部署
```bash
# 安装 Vercel CLI
npm install -g vercel

# 登录
vercel login

# 部署（在项目目录执行）
vercel

# 后续更新
vercel --prod
```

也可以直接在 Vercel 官网（https://vercel.com）连接 GitHub 仓库自动部署。

## 三、开发流程

### 启动开发服务器
```bash
cd melody-lab
npm install      # 安装依赖（首次）
npm run dev      # 启动开发服务器
# 浏览器访问 http://localhost:3000
```

### 构建生产版本
```bash
npm run build    # 构建
npm run start    # 启动生产服务器
```

### 代码审查流程
1. 使用 AI 工具（Codex/Trae/Cursor）打开项目
2. 选择"Code Review"功能
3. 查看 AI 给出的优化建议
4. 选择性采纳并修改

## 四、浏览器调试技巧

1. **打开开发者工具：** F12 或 Ctrl+Shift+I
2. **Console 面板：** 查看 JavaScript 错误和日志
3. **Network 面板：** 检查 API 请求和响应
4. **Elements 面板：** 调试 HTML/CSS 布局
5. **Performance 面板：** 分析页面性能瓶颈

## 五、常见问题解决

### npm install 失败
```bash
# 清除缓存重试
npm cache clean --force
npm install
# 或使用镜像源
npm config set registry https://registry.npmmirror.com
```

### 端口被占用
```bash
# 指定其他端口
npm run dev -- -p 3001
```

### Git 提交冲突
```bash
git pull origin main   # 先拉取最新代码
# 解决冲突后
git add .
git commit -m "fix: resolve merge conflict"
```
