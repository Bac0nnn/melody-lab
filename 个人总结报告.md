## 一、项目概览

**Melody Lab** 是一个基于 Web Audio API 的交互式音乐理论学习工具。用户可以通过交互式钢琴键盘演奏音符，探索和弦与音阶的构成，并通过实时 Canvas 可视化观察音频波形与频谱。项目采用 Next.js 15 + TypeScript 技术栈，共包含 3 个前端页面和 3 个 API 接口。

## 二、遇到的最大困难及解决过程

### 困难 1：钢琴键盘的黑键定位

在实现钢琴键盘时，黑键需要精确地"嵌入"在白键之间。不同八度的黑键位置需要根据其半音位置动态计算，且垂直方向要浮在白键之上。这是整个项目中最具挑战性的 UI 问题。

**解决过程：** 我首先分析了标准钢琴键盘的物理布局规律——每八度 7 个白键和 5 个黑键，黑键在 C#/D#（第 1/3 半音位）、F#/G#/A#（第 6/8/10 半音位）。然后使用 AI 协助我实现了一个基于半音位置到 CSS left 百分比的计算函数，通过绝对定位将黑键精确放置。AI 在这个过程中提供了核心的定位算法，我则负责了视觉样式微调和触摸设备适配。

### 困难 2：Web Audio API 的交互设计

浏览器音频 API 存在一些限制，如用户交互后才能激活 AudioContext，以及快速连续点击时音频节点的管理。

**解决过程：** 我设计了 MusicEngine 单例类，统一管理 AudioContext 的生命周期。每次播放音符时检查 context 状态，suspend 时自动 resume。针对快速点击问题，AI 在后续的 code review 中建议了并发数控制方案。所有音频操作都包含在 try-catch 中，确保在音频不可用的环境下页面仍能正常工作。

### 困难 3：Canvas 可视化性能优化

音频可视化需要每帧刷新 Canvas，在不加优化的情况下，每帧都计算和绘制会导致 CPU 占用过高。

**解决过程：** AI 建议使用 requestAnimationFrame 和 devicePixelRatio 自适应，人工在此基础上添加了 HSL 色彩渐变和阴影效果以提升视觉体验。后续通过 AI CodeReview 进一步提出了使用 ResizeObserver 替代每帧 resize 的建议。

## 三、对 AI 辅助编程的心得体会

这次实训让我深刻体会到 AI 辅助编程在以下方面的巨大价值：

1. **加速原型开发：** 过去需要大量查阅文档和 API 参考的工作（如 Web Audio API 的配置参数、Canvas 绘图 API 的使用），现在可以直接通过自然语言描述需求，由 AI 生成可运行的代码。

2. **降低试错成本：** 在实现钢琴键盘布局时，我尝试了多种方案，AI 能快速给出不同方案的对比和推荐，节省了大量试错时间。

3. **代码质量提升：** AI 不仅能生成代码，还能主动提供优化建议（如 devicePixelRatio 适配、单例模式、性能优化等），这些建议对于初学者来说是宝贵的学习材料。

4. **文档自动化：** AI 协助生成的 Prompt 日志和 API 文档，质量高且结构清晰，大幅度减少了文档编写的工作量。

## 四、个人在项目中的具体贡献

本次项目为个人独立完成，主要贡献包括：

- **架构设计：** 项目目录结构、组件划分、路由规划
- **UI/UX 设计：** 视觉风格、交互流程、响应式适配
- **音乐理论数据：** 和弦/音阶数据的准确性和完整性校对
- **代码审核：** AI 生成代码的审查、修改和集成
- **文档撰写：** README、API 文档、总结报告的编写
- **部署方案：** 零依赖 HTML 版本的构建（确保即使 npm 不可用也能运行）

## 五、反思与展望

1. 增加录音和回放功能
2. 添加完整的 88 键钢琴键盘
3. 实现 MIDI 输入设备支持
4. 增加 AI 驱动的音乐生成功能（根据情绪自动生成和弦进行）

通过这次实训，我不仅掌握了 Next.js 全栈开发和 Web Audio API 的使用，更重要的是学会了如何高效地与 AI 协作编程——这不仅提升了开发效率，也让我对编程的理解从一个"写代码的人"转变为一个"设计解决方案的人"。
